Introduction

Bronson is an opinionated module application framework used for the building of large scalable web applications. Bronson is a thin API wrapper on top of RequireJS providing a simple interface for the loading, unloading, starting and stopping of modules. These modules can bootstrap whatever application framework you desire to build that module in be it custom, Backbone, AngularJS, etc.

In addition to the module bootstrapping layer. Bronson provides a fast, managed, framework agnostic publish/subscribe framework for inter module and application wide messaging. By adhering to Bronson's event namespacing your messages will be easily maintainable and more importantly easily understandable(readable) to your team.

Bronson's Publish/Subscribe

Simple contrived example

{% highlight coffeescript %} # Module Foo # # we subscribe by providing the subscriber, channel and finally topic Bronson.subscribe 'foo:app:greeting', (data) -> console.log data.message ... # Module Bar # # we publish by providing a channel name followed by topic Bronson.publish 'app:greeting' message: 'hello!' # we can then easily remove any subscription by doing the following Bronson.unsubscribe 'foo' {% endhighlight %}

Features

  • Module loading & unloading through Bronson.Core
  • Managed publish & subscribing of events through Bronson.Core & Bronson.Permissions

Dependencies
